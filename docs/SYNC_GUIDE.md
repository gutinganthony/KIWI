# 兩台機器同步說明書（給 Jake 看的白話版）

> 你在公司用 **Mac mini**，回家用 **MacBook**。
> 這份文件教你怎麼讓兩台電腦永遠看到「同一個 KIWI」。
> 不用懂技術名詞，照著做就好。

---

## 一句話原理

GitHub 上有一份「正本」KIWI。
你的兩台 Mac 各自有一份「副本」。

- **開工前**：把正本的最新內容「抓下來」→ 你的副本就跟正本一樣新
- **收工前**：把你今天改的東西「送上去」→ 正本被更新

只要養成這個習慣，兩台機器永遠一致，不會打架。

---

## 你每天只要記兩個動作

### 🌅 開工前（坐下來第一件事）

打開 Claude Code，跟它說：

> **「幫我 pull 最新的」**

（pull = 把正本抓下來。Claude 會自動執行，你不用記指令。）

### 🌙 收工前（離開前最後一件事）

跟 Claude 說：

> **「幫我把今天的改動 push 上去」**

（push = 把你的改動送回正本。）

---

## 就這樣，記住這個節奏：

```
到公司 Mac mini  →  「pull 最新的」 →  開始工作 →  「push 上去」 →  下班
回家 MacBook     →  「pull 最新的」 →  開始工作 →  「push 上去」 →  睡覺
```

> 🔑 **黃金法則：開工先 pull，收工先 push。**
> 只要遵守，兩台機器就不會有「我這台的版本比較舊」的問題。

---

## 萬一忘記了會怎樣？

- **忘記 pull 就開始改**：你可能在舊版本上工作，等下 push 時 Claude 會提醒你「有衝突」。別緊張，直接跟 Claude 說「幫我處理衝突」，它會幫你合併。
- **忘記 push 就關電腦**：你今天的改動只留在這台 Mac，另一台看不到。下次回到這台 Mac，記得先 push 再換機器。

> 💡 不確定狀態時，問 Claude：**「我現在有沒有沒 push 的東西？」**

---

## Ruflo 的記憶要不要同步？

**不用特別處理。** Ruflo 在每台機器上有自己的「快取記憶」，
但 KIWI 真正的記憶是這個 repo 裡的文件（像 `KIWI_INDEX_FRAMEWORK.md`）。

只要你 pull/push 同步好文件，兩台機器的 Claude + Ruflo 都會讀到同一份知識。
Ruflo 本地的記憶遺失也沒關係，正本永遠在 GitHub。
