# PA team

PA team 的每日報告與週期性摘要存放區。

## 資料夾結構

```
PA team/
├── README.md          # 本說明文件
├── reports/           # 每日報告(每天上傳一份)
└── summaries/         # 每約四週產出的彙整摘要
```

## 工作流程

### 1. 每日上傳報告
- 每天將當日報告放進 `reports/` 資料夾。
- 建議檔名格式:`YYYY-MM-DD.md`(或 `.txt` / `.pdf` / `.docx` 等),例如 `2026-06-05.md`。
- 以日期命名可確保排序正確,也方便摘要時依時間區間擷取。

### 2. 約每四週產出摘要
- 大約每四週(可自行調整),依據 `reports/` 中該區間的報告產生一份摘要。
- 摘要存放於 `summaries/`,建議檔名格式:`YYYY-MM-DD_to_YYYY-MM-DD_summary.md`,
  例如 `2026-06-05_to_2026-07-02_summary.md`。

### 摘要會包含
- **期間概覽**:涵蓋的日期區間與報告數量。
- **重點進度 / 成果**:跨報告彙整的主要事項。
- **趨勢與變化**:重複出現的主題、改善或惡化的指標。
- **風險 / 待辦 / 阻礙**:需要關注或追蹤的項目。
- **下一步建議**:基於整體趨勢的行動建議。

## 如何請我產出摘要

只要告訴我類似:
> 「幫我彙整 PA team 過去四週的報告摘要」

我會讀取 `reports/` 中對應期間的報告,產出摘要並存到 `summaries/`。
你也可以指定區間,例如「彙整 6/5 到 7/2 的報告」。
