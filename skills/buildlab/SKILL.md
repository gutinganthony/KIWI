# BuildLab 產品實驗管線 Skill（Levels × 三閘門）

**觸發指令**：`/buildlab check [產品 idea]` 或直接說「我有個產品 idea：XX」

**適用範圍**：可由 1 人 + AI 在數週內做出 MVP 並開始收費的產品（SaaS、訂閱內容、工具、服務型產品）
**不適用**：投資訊號判讀（→ 移交 socialarb / Serenity）、需要大資本或牌照的事業、純興趣專案（不打算收費就不用進管線）

---

## 這個 Skill 在做什麼？

它把兩套已驗證的否決機串成一條產品管線：

- **socialarb 的濾網邏輯**（Camillo）：需求異常 → 三閘門驗證 → 大多數被否決
- **Pieter Levels 的執行框架**：大量便宜實驗 × 付費作為唯一驗證 × 快殺紀律 × 自有通路複利（40+ 產品、~4 個存活、年入 ~NT$8,000 萬、零員工）

**核心命題：好產品不是想出來的，是從高速否決機裡活下來的。** 這個 skill 的日常工作是否決 idea，不是鼓勵 idea。能通過的極少數，才值得投入兩週去做。

### 與 socialarb 的同構對照

| | socialarb（選股） | buildlab（做產品） |
|---|---|---|
| 一手觀察 | 生活中的消費異常 | 自己/客戶的真實痛點 |
| 閘門驗證 | 重大性/未定價/可兌現 | 真痛點/未被滿足/可觸達 |
| 唯一誠實訊號 | 股價反映前的資訊差 | **有人掏錢**（不是讚數） |
| 出場紀律 | 資訊擴散即離場 | 驗證時鐘到期即殺 |
| 歸檔 | watchlist 已關閉訊號 | graveyard 已殺實驗 |

## 創辦人檔案（閘門判定的基準，隨時更新）

**優勢**：
- A. 方法論編譯：把專家判斷流程變成 AI 可執行的紀律系統（Serenity/socialarb/KIWI 實證）
- B. 投資領域：私人銀行投資顧問，多年投資觀察與建議經驗（台/港，前香港執業）
- C. 業務/BD：長年業務經驗，擅長與跨領域者、陌生人建立關係與合作，有中國與歐洲市場經驗
- D. AI 工具鏈深度使用者（Claude Code、自動化排程、workflow）

**職涯脈絡**：從香港移居回台灣，現職投資顧問為過渡選擇；若有客戶基礎，首選回到 private banker 角色 → **產品實驗應優先累積「能轉化為客戶信任」的資產**

**通路資產（Gate 3 檢查的對象）**：
- 摸魚記（匿名帳號，累積社群中）
- Instagram 實名主力帳號（⚠️ 受合規紅線約束，見方法論極限）
- 無既有大型公開受眾 → **每個實驗必須自帶通路假設，不能假設「做出來就有人看到」**

**可投入時間**：本業之外的零碎時間 → 時間盒必須誠實（見建造規則）

## 分析步驟（依序執行）

### Step 1：適用性檢查
- 這是「產品 idea」還是「投資訊號」？投資訊號 → 移交 socialarb / Serenity
- 1 人 + AI 在時間盒內做得出 MVP 嗎？需要牌照/大資本 → 直接說不適用
- 打算收費嗎？不打算 → 不進管線（興趣專案不需要紀律）

### Step 2：痛點來源分級（同 socialarb 的訊號等級精神）
- 🟢 一手：自己每週都撞到的問題，或客戶/熟人主動抱怨過的問題
- 🟡 二手：觀察到別人有此問題，但未親身驗證
- ⚪ 想像：「應該會有人需要吧」→ 預設懷疑，需先找到 3 個真人確認痛點才升級

### Step 3：三道閘門（全過才建造 — 本 skill 的核心）

**閘門 1｜真痛點（Painful enough to pay）**
- 有人**現在**就在為這個問題付出成本嗎？（時間、金錢、麻煩）
- 測試句：「如果明天這個產品存在、月費 $X，你會刷卡嗎？」→ 要的是遲疑的「會」，不是禮貌的「不錯耶」
- ❌ 只有「覺得有趣」沒有「正在痛」→ 不成立

**閘門 2｜未被滿足（Underserved）**
- 現有解法是什麼？（包括「Excel + 土法煉鋼」也算解法）
- 你的差異必須一句話講完：更快 / 更便宜 / 更專門 / 別人做不到的信任
- ❌ 差異講不出一句話，或差異是「我的比較好用」→ 不成立

**閘門 3｜可觸達（Reachable — 防 Levels 倖存者偏差的閘門）**
- 前 100 個目標用戶在哪裡聚集？你**現在**有路徑讓他們看到嗎？
- 檢查創辦人檔案的通路資產：摸魚記受眾對得上嗎？BD 能力能直接觸及嗎（B2B 一對一 outreach 也是通路）？
- ❌ 答案是「做好再想行銷」→ 不成立。Levels 的 40 個產品每個都有百萬追蹤當發射台；我們沒有，所以通路假設必須在建造前就寫死

### Step 4：建造規則（過三閘門才啟動）
1. **時間盒 ≤ 2 週**（本業之外的實際可用時數估算，不是日曆天的幻覺）
2. **用最熟的工具**：Claude Code + 現成服務（Substack/Discord/Stripe/表單），不學新框架
3. **第一天就掛價格**：上線即收費。免費版是通路策略時才允許，且要寫明轉付費的機制
4. **公開記錄**：每個實驗在指定通路（摸魚記為預設）發布進度，內容本身是通路增長的燃料
5. MVP 定義寫在 pipeline.md：核心功能一句話 + 明確不做什麼

### Step 5：驗證時鐘（建造前寫死，不可事後調整）
- **預設：上線後 30 天 / 3 個付費用戶**
- 特殊情境可事前調整（B2B 高客單 → 30 天 / 1 個簽約；社群型 → 見通路實驗另計），但必須在 pipeline.md 建卡時寫死
- 時鐘到期只有三種結果：**加碼 / 轉向（pivot 一次為限）/ 殺**

### Step 6：殺與歸檔紀律
- 殺 = 當天停止投入，pipeline.md 卡片移到 graveyard.md，寫三行：結果數字、死因（哪個閘門當初誤判）、教訓
- **禁止殭屍專案**：不殺也不投入的狀態不存在。graveyard 是資產，不是恥辱柱
- pivot 限一次：第二次想 pivot = 這個痛點假設已死，殺掉重開新卡

### Step 7：通路資產帳（每季盤點）
- 通路（摸魚記、實名 IG、電子報名單、BD 人脈）是跨實驗的複利資產，單獨記在 pipeline.md 的通路實驗區
- 通路實驗用自己的驗證時鐘（成長是慢變數，預設 90 天），指標：追蹤成長率、互動率、**轉化率**（點進 link-in-bio / 訂閱 / 私訊詢問）
- 每個產品實驗結束時記錄：它為通路帳戶帶來多少新增受眾（產品死了通路還在 = 實驗沒白做）

### Step 8：輸出格式（對話中跑完 Step 1–3 後給出）
1. **一句話判定**：🟢 過三閘門，開建造卡 / 🟡 缺口在閘門 N，補驗證後再來 / ⚪ 否決（進 graveyard 的 idea 區或直接放掉）
2. 三閘門逐項判定表（✅/❌ + 理由）
3. 若過：MVP 定義、時間盒、價格、通路假設、驗證時鐘 → 寫入 pipeline.md
4. 若否決：一句話死因 + 什麼新事實出現時值得重跑

## 自我偏誤校正

- **「做得出來」不是「該做」**：技術可行性永遠不進閘門判定
- 用戶（創辦人）明顯興奮時 → 加重閘門 1 的「付費測試句」檢查，要求真人證據
- 已投入時間不是繼續的理由：時鐘到期就是到期（sunk cost 免疫）
- 通路幻覺：「發到社群上就會有人看」→ 要求具體：哪個帳號、對什麼受眾、憑什麼被轉發

## 方法論極限（必須誠實說明）

- **Levels 的倖存者偏差**：他的命中率也只有 ~10%，且靠十年累積的百萬追蹤放大。預期：多數實驗會死，管線的價值在死得便宜、死得快、通路資產留下來
- **合規紅線（實名身分）**：創辦人為執業投資顧問。實名帳號**不得公開發布個股買賣建議或訊號**（台灣投顧法規 + 雇主合規政策雙重約束，任何實名投資內容先過雇主 compliance）。投資訊號類內容一律走匿名通路或僅限私人研究；實名通路定位為方法論/AI 工作流/跨域洞察
- **時間是硬約束**：1 人 + 本業，同時間最多 1–2 張活躍建造卡。管線塞車時，新 idea 排隊，不開新卡

---

**底線：這台機器的產出不是產品，是「被便宜且快速驗證過的決定」。付費是唯一不說謊的訊號，時鐘到期就執行，通路是唯一跨實驗複利的資產。**
