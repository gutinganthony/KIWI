# Serenity 供應鏈瓶頸選股 Skill

**觸發指令**：`/serenity analyze $TICKER` 或 `/serenity analyze [產業/主題]`

**適用範圍**：AI 基礎建設、半導體、光模組供應鏈中的小市值上游股  
**不適用**：消費性股票、生技、純軟體、大型市值已被充分定價的公司

---

## 這個 Skill 在做什麼？

它複製 Twitter 分析師 Serenity（@aleabitoreddit）的選股邏輯：
**不問「這支股票好不好買」，而是問「AI capex 的錢最終流向哪個被市場忽視的上游瓶頸？」**

Alpha 來自於找到機構投資人還沒畫出的多層供應鏈關係。

---

## 分析步驟（依序執行）

### Step 1：確認適用性
- 是否在 AI 基礎建設超週期（超算、光通訊、HBM 記憶體、AI 電源）的範圍內？
- 不在範圍內→直接說「這不是我的射程」，不強套框架

### Step 2：鎖定終端需求
- 找到具體的硬體需求（GB300、TPU pod、1.6T 光模組）而不是模糊的「AI 成長」
- 確認時間軸：6–18 個月內有沒有需求放量的觸發點

### Step 3：畫 6–9 層供應鏈（從下游往上追）
```
AI 超算 → 設計贏家（Nvidia/Broadcom）→ 關鍵元件 → 子組件 → 上游材料 → 原料
```
每一層標註：已知供應商 / OSINT 推論 / 純推測（三個等級要分清楚）

### Step 4：找最深的瓶頸
- 哪一層有「真正的供給約束」？（不是產能緊張，是結構性壟斷）
- 地理是否加劇稀缺？（例：銦 70% 來自中國）
- 認證週期是否讓財報落後實際需求？

### Step 5：對照 14 個篩選條件
**硬性必要條件（缺一票否決）：**
- 壟斷或雙頭壟斷結構（有定價權）
- 市值 <$2B vs 下游數兆美元 TAM
- 多客戶 design-in（不是單一客戶）
- 資產負債表能撐過放量爬坡期

**加分條件（越多越好）：**
- 認證週期長→收入滯後→目前財報「看起來弱」但實際訂單已確認
- 主要客戶已開始預購產能
- 國家安全護城河（美國製造優勢）
- backlog 可見度高

### Step 6：掃描 10 個紅旗（有一個就降評）
- 無限制 ATM 增資（直接否決）
- 單一客戶佔比 >50%
- 中國曝險（原則性排除）
- 零收入純炒作
- 技術實現時點 >2028
- 毒性債務超過存活跑道

### Step 7：估值（依公司類型選工具）

| 公司類型 | 估值方法 |
|----------|---------|
| 供給約束瓶頸股 | 產能即收入 + 同層級同業市值對比 |
| 高成長（未來收入） | 預估 ARR / 市值；向前看 P/E（2027–28 盈利能力） |
| 深度價值 | P/E、EV/FCF、淨現金佔市值比例 |

**門檻參考**：
- 個位數 forward P/E = 極度便宜
- 法人持股 <40% = 上漲空間未充分釋放
- 淨現金 ≈ 市值 = 下行保護

### Step 8：催化劑時間表
不等財報，用領先指標驗證：
- 上游商品現貨價（例：7N 銦現貨價在 SMM 的走勢）
- 同業財報電話會議交叉驗證
- 客戶 capex 公告
- 出口管制政策日期
- 指數成分調整

### Step 9：輸出報告

**單一股票報告格式：**
1. 一句話結論（含方向：🔥Fire Sale / 極強力買進 / 買進 / 持有 / 賣出）+ 時間窗口
2. 事實基礎（現有數據，標明來源：10-K / 10-Q / IR）
3. 分析主體：供應鏈逆向追蹤、瓶頸判斷、估值、催化劑時間表
4. 否證條件：什麼情況代表論點已經錯了？

**來源標示原則（每個數字都要標）：**
- `已確認`：有一手文件可引用的
- `[推論]`：查過但找不到一手來源，說明查找努力
- `[推測]`：無證據，只用於定性定位，不能用於數字結論

### 美股風險分級（beta × 市值）——美股標的輸出必附

凡出現在報告、週報、儀表板的**美股**標的，一律附 `風險：高/中/低`。
兩維各自分級後**取較高風險者**（保守原則）：

| 維度 | 低 | 中 | 高 |
|---|---|---|---|
| Beta（5 年月線 vs S&P 500，Yahoo 口徑） | ≤ 0.9 | 0.9–1.3 | > 1.3 或**查無** |
| 市值 | ≥ $10B | $2B–$10B | < $2B |

- 例：beta 0.8 但市值 $800M → **高**；beta 1.1、市值 $15B → **中**。
- 兩維的理由：beta 量測對大盤系統性波動的敏感度；市值是流動性、財務韌性、
  資訊覆蓋度的 proxy（採標準 large/mid/small cap 分界 $10B / $2B）。
- **備註義務**：任何顯示風險等級的地方（digest 表、儀表板）都要在下方附一行
  說明本分級法，讓讀者不用回來翻這份檔案。
- Serenity 專獵 <$2B 上游卡口 → 篩出的美股**多數天然是「高」**。此標注用於
  倉位 sizing 與預期管理，**不是否決訊號**（進出仍由觸發價/否證條件決定）。
- 取數：`python3 projects/avi-v5/scripts/quote.py --risk TICKER...`（自動拉
  beta+市值並判級）；API 不通時用搜尋 snippet 交叉驗證後照表判級並標來源。
- 非美股暫不分級（beta 基準指數不同），欄位填 `—`。

## 自我偏誤校正

- **風險/熊市情境永遠先寫**，牛市情境後寫
- 用戶明顯想買時→主動加重熊市論點和弱點分析
- 論點依賴 [推論] 或 [推測] 時→下調信心等級

---

## 方法論極限（必須誠實說明）

這套框架在以下情況**失效**：
- 成熟大型股（已被市場充分定價）
- 純軟體（沒有實物供給約束）
- 消費性、生技、宏觀選股

失效時直接說「這不適合用 Serenity 框架分析」，不要強套。
