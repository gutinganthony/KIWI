# 社會套利觸發訊號 Skill（Social Arbitrage）

**觸發指令**：`/socialarb analyze $TICKER` 或 `/socialarb check [我觀察到的現象]`

**適用範圍**：消費品、餐飲、服飾、娛樂、消費性 App、小到中市值、「看得到摸得到」的產品公司
**不適用**：B2B 深供應鏈上游、大型市值股（訊號被稀釋）、生技新藥、純宏觀選股

---

## 這個 Skill 在做什麼？

它複製散戶傳奇 Chris Camillo 的「社會套利」（Social Arbitrage）邏輯：

**不問「這支股票財報好不好」，而是問「現實世界已經發生、但華爾街還沒定價的事實是什麼？」**

Alpha 來自資訊差的時間窗口：**買在資訊差確認的當下，賣在資訊擴散為市場共識的時候。**

Camillo 的績效由 Jack Schwager 在《Unknown Market Wizards》查核過（2006 年起每月券商對帳單，年化約 68–77%，同期 S&P 500 僅 7.6%）。這套 skill 把他的直覺流程「機械化」成可否證的閘門。

### 與 Serenity 的分工

| | Serenity | 社會套利 |
|---|---|---|
| 找什麼 | 結構性**供給**約束 | 需求端**突發異常** |
| 變數速度 | 慢（6–18 個月） | 快（數週–數月） |
| 訊號源 | 供應鏈逆向追蹤、商品現貨價 | 社群聲量、通路缺貨、KOL 事件 |
| 標的 | 小市值 B2B 上游 | 消費性 B2C |

兩者互補：Serenity 是「錢流向哪個瓶頸」，社會套利是「哪個需求先爆」。

## 分析步驟（依序執行）

### Step 1：適用性檢查
- 這是不是「消費者能直接觀察到」的產品？（App、飲料、鞋子、玩具、餐廳）
- 公司市值夠小、單一產品能動搖營收嗎？
- 不適用（B2B、大型股、生技）→ 直接說「這不是社會套利的射程」，若指向供應鏈上游則移交 Serenity，不強套框架。

> **主題 ≠ 訊號**：若用戶給的是宏觀趨勢（「大家開始注意 X」）而非具體急性異常，先判斷這是**慢變數 megatrend** 還是**急性 spike**。megatrend 通常閘門 2 主題層級已破（公開共識），且純受惠者常為私人公司（無 ticker）→ 不進場，改記入 watchlist 的「主題掃描筆記」，並定義「要等哪個收斂後的急性訊號才回來跑三閘門」。收斂方向：某上市品牌單一新品急性缺貨且股價未動、私人純玩家 IPO/被併（event-driven）、單一事件引爆單一 SKU 搶購。

### Step 2：訊號來源分類（六類，標明訊號的「原生地」）
1. **社群聲量異常**：TikTok 標籤、YouTube 開箱、Reddit 討論串、X 話題、Google Trends 斜率突變
2. **實體通路異常**：缺貨、排隊、貨架空間變化（Camillo 的 Snapple 賣權、Elmer's 膠水原型）
3. **KOL／單一事件催化**：爆紅影片、名人帶貨（e.l.f. 遮瑕膏被 Jeffree Star 開箱，半年 $8→$17.5）
4. **行為結構改變**：疫情、法規、天氣、新平台崛起（Crocs 居家熱、Dorel 自行車缺貨 +629%）
5. **生活圈一手觀察**：自己或身邊人的消費行為突變（最高價值、最難被複製的訊號）
6. **反向訊號**：熱門產品突然退燒（用於做空或迴避，Snapple 貨架被擠掉即此類）

> 標註訊號等級：`一手觀察` > `數據可交叉驗證` > `二手轉述`。訊號源越接近第一線，可信度越高。

### Step 3：三道閘門驗證（全過才成立 — 這是本 skill 的核心）

**閘門 1｜重大性（Materiality）**
- 這件事若為真，能不能推動該公司營收 ≥ 5–10%？
- 要查：該產品／品牌佔公司總營收比重、公司市值大小（越小越敏感）
- ❌ 若該產品只佔營收 2%，再紅也動不了股價 → 不成立

**閘門 2｜未定價（Not-yet-priced）**
- 華爾街知道了嗎？逐項檢查：
  - [ ] 近期股價尚未異動（沒有先跑）
  - [ ] 賣方分析師報告未提及
  - [ ] 主流財經媒體未報導
  - [ ] 聲量起點距今仍在早期（時間差還在）
- ❌ 任一項顯示已擴散 → 資訊差已消失，不成立

**閘門 3｜可兌現（Catalyst-reachable）**
- 距離下一個驗證點（財報、月銷數據、開店數）多久？
- 資訊差有沒有足夠時間在窗口內轉化為股價？
- ❌ 若催化劑要等 18 個月，時間成本過高、變數太多 → 降級觀察

### Step 4：訊號強度分級

| 等級 | 條件 |
|---|---|
| 🟢 **S 級** | 一手觀察 + 數據可交叉驗證 + 三閘門全過 |
| 🟡 **A 級** | 數據異常明確但缺一手驗證，或閘門 2 有部分擴散 |
| ⚪ **B 級** | 有趣但未達標 → 進 watchlist 繼續追蹤，不進場 |

### Step 5：進出場邏輯（與價格目標無關）

- **進場** = 三閘門確認的當下，**不等回檔**。資訊差是時間敏感資產，每天都在衰減。
- **出場** = 資訊擴散訊號出現任一項：
  - 財報公布並反映了這個趨勢
  - 賣方分析師開始寫這個故事
  - 主流媒體開始報導
  - 社群聲量見頂回落（Google Trends 斜率轉負）
- **核心紀律**：優勢來自私有資訊，當它變成公開共識，優勢即消失 → **立刻離場，無論賺賠**，不戀棧、不凹單。

### Step 6：否證條件（每個訊號建立時必寫）
- **曇花一現（fad decay）**：聲量斜率在 2 週內轉負 → 需求不持續
- **聲量沒轉化為銷售**：信用卡數據／通路檢查對不上聲量 → 訊號是假的
- **訊號已被定價**：進場後回查發現股價／報告早已反應 → 閘門 2 誤判，認錯出場
- **重大性誤判**：深查後發現爆紅產品佔營收比重太低 → 閘門 1 誤判

### Step 7：部位管理（明確不照抄 Camillo）

> ⚠️ Camillo 用**集中持股 + 選擇權槓桿**，那是他極高風險偏好下的選擇，有強烈倖存者偏差。同樣方法在判斷錯誤時會巨額虧損。本 skill 內建保守上限：

- 單一社會套利訊號部位上限：建議 ≤ 總資產 5%
- 選擇權：僅限 🟢 S 級訊號，且投入金額以「歸零也可接受」為上限
- 同時持有的社會套利部位總和設一個上限（例如 20%），避免整組同時踩雷

### Step 8：輸出報告格式（沿用 Serenity 慣例）

1. **一句話結論** + 方向（🟢進場 / 🟡觀察 / ⚪不足 / 🔴反向做空）+ 訊號等級 + 時間窗口
2. **事實基礎**（每個數字標源）：
   - `已確認`：有一手數據可引用（Google Trends 截圖、公司營收結構、通路照片）
   - `[推論]`：查過但無一手來源，說明查找努力
   - `[推測]`：無證據，僅定性定位，不用於數字結論
3. **三閘門逐項判定**（重大性 / 未定價 / 可兌現，各給 ✅/❌ + 理由）
4. **觸發 / 否證表**（進場觸發、出場訊號、否證條件）
5. **驗證時鐘**：下一個能驗證訊號的日期（財報日、月銷數據、開店公告）

### Step 9：與 KIWI 現有框架的接口
- **疊加 WaveTrend**：訊號三閘門成立 + WT 超賣 → 進場時點最佳（見 `skills/wavetrend/SKILL.md`）
- **移交 Serenity**：若社會套利訊號指向的是供應鏈上游 B2B 瓶頸 → 改用 Serenity 深掘（見 `skills/serenity/SKILL.md`）
- **ACT index 判讀**：訊號的持續性可對照 AVI/CRI/TSI（見 `docs/KIWI_INDEX_FRAMEWORK.md`）

## 自我偏誤校正

- **先寫「這訊號為什麼可能是假的」**（否證條件），再寫多頭論點
- 用戶明顯很興奮想追時 → 主動加重閘門 2（是否已定價）的檢查
- 論點依賴 `[推論]` 或 `[推測]` 時 → 下調訊號等級
- 「聲量 ≠ 銷售 ≠ 股價」永遠是預設懷疑，三者要分別驗證

## 方法論極限（必須誠實說明）

這套框架在以下情況**失效**：
- B2B、深供應鏈、無法被消費者直接觀察的產品
- 大型市值股（單一產品訊號被整體營收稀釋）
- 資訊擴散極快的今日環境（Reddit、量化基金也在掃社群數據，窗口比 2007 年窄）
- 生技、宏觀、利率驅動的標的

失效時直接說「這不適合用社會套利框架分析」，不要強套。此外，這是**極度耗時的主動策略**（Camillo 每天掃社群數據、還做了 TickerTags 輔助），不是被動投資。

---

**底線：社會套利把市場缺口視為「資訊擴散的時間差」，需要一手觀察 + 三閘門驗證，不是追熱點。Alpha = 在華爾街定價之前，用消費者的第一線身份看到需求異常。**
