# 像量化交易台一樣做模擬 — 從銅板到機構級模擬引擎（預測市場為案例）

> **原文**：X（Twitter）@gemchange_ltd〈How to Simulate Like a Quant Desk. Every Model, Every Formula, Runnable Code〉（2月28日長文，附可執行 Python）
> **收錄日期**：2026-07-14
> **一句話**：一條「散戶直覺 → 機構級模擬堆疊」的教學路線，拿 Polymarket 預測市場當貫穿案例，八個部分層層疊加：Monte Carlo → 稀有事件 → 即時濾波 → 變異數縮減 → 尾部相依 → agent 模擬 → 生產系統。

## 1. 八個部分各在幹嘛

| Part | 主題 | 核心訊息 |
|---|---|---|
| I | 銅板謬誤 | 散戶把 0.62 的合約當「已知偏差的銅板」下注。真實合約有幾十個參數：你對自己估計的信心、新資訊該怎麼更新、與其他合約的相關性、到期前價格路徑能不能讓你獲利出場 |
| II | Monte Carlo 基礎 | 收斂率 O(N^-1/2)；**p=0.5 時變異數最大**（最熱門的合約統計上最難估準）。附 GBM 二元合約定價 code。**Brier score** 校準評分：<0.20 好、<0.10 優秀、頂級選舉預測（538）0.06–0.12 |
| III | 稀有事件 | 0.003 的尾部合約用 10 萬條 crude MC 只會出現 0 或 1 次命中＝無用。**Importance sampling／exponential tilting**：把分布往稀有區傾斜再用概似比校正，變異數降 100–10,000 倍 |
| IV | 即時更新 | **粒子濾波（Sequential MC）**：把市場價當「帶噪音的觀測」而非真值，濾波器平滑單筆成交的跳動、傳遞不確定性（選舉夜逐州更新案例，附完整 class） |
| V | 變異數縮減三招 | Antithetic（對稱翻轉，省 50–75%）＋ control variate（用 BS 封閉解當基準）＋分層抽樣。可疊加，常規 100–500x——「生產環境的基本盤」 |
| VI | 尾部相依 | **Gaussian copula 尾部相依＝0（2008 年的教訓）**；t-copula(ν=4, ρ=0.6) 尾部相依 ≈ 0.18。搖擺州「全掃」機率 t-copula 比 Gaussian 高 2–5 倍。>5 檔合約用 vine copula（pyvinecopulib） |
| VII | Agent-based 模擬 | **零智能交易者也能讓市場近乎有效**（Gode & Sunder 1993；Farmer 2005 用一個參數解釋 LSE 96% 的價差橫斷面變異）。informed/noise/MM 三類 agent 的互動決定價格收斂速度 |
| VIII | 生產堆疊 | 五層：資料接入（CLOB WebSocket/新聞 NLP）→ 機率引擎（貝氏階層＋粒子濾波＋jump-diffusion ensemble）→ 相依建模（vine copula/因子/尾部）→ 風險管理（EVT VaR/反向壓測）→ 監控（**Brier 追蹤、PnL 歸因、模型漂移偵測**） |

## 2. 可信度評估（誠實）

- **數學內容是標準教科書級**，參考文獻是真實經典（Kyle 1985、Glosten-Milgrom 1985、Gode & Sunder 1993、Aas et al. 2009 vine copula、Merton 1976 jump-diffusion…）——內容本身不是騙局。
- **但這是 X 內容行銷**：與 07-10 解剖過的 Polymarket 導流文同一個生態圈。code 是教學玩具（GBM 假設、ABM 極簡化、無交易成本/滑價），**不是可直接上線的系統**。當課綱用，別當聖杯。
- 適用邊界：這套方法假設你能寫下資料生成過程；它解決「算得準」，不解決「模型對不對」——對 KIWI 而言後者永遠是主要矛盾。

## 3. KIWI 怎麼運用

### 直接可用
1. **尾部相依 → 部位管理鐵律**：持股（宇瞻/AAOI/村田/JEM/MU 期貨）全在同一條 AI 供應鏈。Part VI 的教訓翻成白話：**極端情境下它們不是五個分散的注，是一注**（尾部相依遠高於平時相關係數的暗示）。壓測基準情境應該是「全部一起跌」；「AI 供應鏈總曝險」要當單一部位管理——正好呼應 Serenity 批次紀律與 LFI≥95 謹慎規則。
2. **Brier score → 四錶校準追蹤**：把「LFI≥95 → 未來 3 個月 ≥10% 修正」「CRI HIGH → 23-42 天崩盤」這類警訊化成機率預測，逐季算 Brier/命中率，制度化回答「我們的錶準不準」。（低成本：一支腳本＋每季一行紀錄。）
3. **粒子濾波思想 → LFI 讀數判讀**：LFI 日間噪音大（兩週內 2→74→27），單日讀數是「帶噪音的觀測」不是真值。dashboard 剛上的「連續維持天數」就是這個思想的簡化版——**看持續狀態、不看單點**。若未來要升級：5 日中位數平滑或輕量濾波。

### 概念收穫（不必寫 code）
4. **稀有事件謙卑**：「一週 −20%」這種問題，歷史樣本本來就只有個位數次——這解釋了為什麼 KIWI 堅持事件研究＋長樣本＋block bootstrap，而不是套參數模型硬算（假設錯了，算得再精確也是錯的精確）。
5. **p=0.5 變異數最大**：對半開的爭論（最熱門、最吵的）恰好統計上最難估準。研究時間的性價比：極端定價的情境 > 50:50 的熱門辯論。
