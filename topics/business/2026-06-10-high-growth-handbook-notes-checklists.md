> 讀書筆記：矽谷創業者/投資人 Elad Gil 的規模化實戰手冊。全書免費：https://growth.eladgil.com/
> 用途：給 Redefine Fund 籌備 + 用人/治理參考。含兩份可直接用的 checklist。

## 全書一句話
**新創從「靠創辦人個人努力」過渡到「靠系統與制度」的實戰手冊。** 作者刻意寫得「痛苦地具體」，避開沒開過公司的人講的空話。核心警告：**某些問題在規模化的關鍵階段不處理，等公司更大再處理會貴上十倍。**

## 五大主題速覽
1. **CEO 的角色**：無法外包的只有 4 件——①設定願景/方向/策略 ②招到並留住最頂尖的人 ③確保公司有錢 ④訂營運節奏與標準；其餘授權。CEO 要「自我顛覆」，每階段升級自己；願景要重複講很多次；管理好自己的孤獨與時間。
2. **管理董事會**：5–6 人為上限；早期投資人 ≠ 好董事；會議大部分時間留給策略；董事會是工具不是老闆。
3. **招募/用人/組織**（全書最重）：招「能帶你到下一階段」的高管；招募流程標準化；用錯人要快砍；reorg 24 小時內完成；隱性文化要明文化。
4. **產品與成長**：找到 PMF 後重心從「產品」轉「通路/分發」；第二個成功產品更難但不能停。
5. **募資/估值/併購/IPO**：後期募資 + secondary 讓人套現；估值過高反成包袱；M&A 常是買團隊/買時間；IPO 想清楚「為什麼上市」。

## ✅ Checklist A：招募高階主管（VP/C-level）

### 1. 招募前——先定義成果，不是寫 JD
- [ ] 寫出這職位未來 **12–18 個月要達成的 3–5 個具體成果（scorecard / outcomes）**，不是職責清單。
- [ ] 釐清「**現在這階段**需要的人」vs「兩階段後需要的人」→ 招能帶你到**下一階段**的，不是名氣最大的。
- [ ] 定 level，避免 over-hire：招過慣管 500 人的人來管 5 人，常會水土不服。
- [ ] 跟共同創辦人/合夥人**對齊這個角色的決策權邊界**（避免日後權責打架）。

### 2. Sourcing
- [ ] CEO **親自 sourcing** + 動用投資人/董事/人脈；高管極少靠 inbound。
- [ ] **一次面好幾位做 calibration**，不要看到第一個「還行」就停。

### 3. 面試——標準化 + 看實戰
- [ ] **每位候選人走同一套關卡、同一套評分表**（職能 / 領導 / 價值觀 / 跨部門協作）。把招募 SOP 訂得比其他授權更細，因為每個面試官都帶偏見。
- [ ] 深挖「**相似規模 / 相似成長階段**」的實戰：請他講**具體決策、數字、團隊從 X 人帶到 Y 人**的過程——不要聽抽象哲學。
- [ ] 給**情境題/作業**（90 天計畫、實際問題拆解），看思路不是看話術。
- [ ] 問「**離開前公司的真正原因**」「親手做過最難的決定 + 失敗」→ 看 ownership 還是甩鍋。
- [ ] 測「**帶得動比他資深的人嗎？跨部門協作如何？**」

### 4. Reference check（最重要、也最常被草草帶過）
- [ ] 做 **back-channel 背靠背查核**：不只他給的 reference，**找你自己人脈裡跟他共事過的人**。
- [ ] 三個視角都問：**前主管(向上)、前部屬(向下)、平行同事(橫向)**。
- [ ] 問具體狠問題：「**你會再雇他嗎？**」「他最大的弱點？」「**壓力下他怎麼樣？**」「他帶的人**有多少跟著他走**？」
- [ ] 注意：**不願意給你某些 reference** 是紅旗。

### 5. 成交 + Onboarding
- [ ] 賣**願景 + 真正的舞台(scope/impact)**——高管要的不只是錢。
- [ ] 設清楚 **30/60/90 天期望 + 成功指標**；前 90 天高頻 1:1、引介關鍵關係。
- [ ] 設「**快速失敗檢查點**」：3–6 個月明確 review，**錯了要快砍**（拖越久越貴、越傷團隊）。

### 🚩 紅旗
- 只有大公司品牌、沒有「從小做大」的實戰｜講得出策略、講不出自己親手做的事｜reference 含糊或閃避｜你只是「面起來舒服」就想招。

## ✅ Checklist B：管理董事會

### 1. 組成（一開始就設計好）
- [ ] 席次**控制在 5–6 人**（含創辦人）；越多越政治化、越難開。
- [ ] **早期投資人不一定是好董事**——逐位評估「對**下一階段**的實際幫助」。
- [ ] 設計**創辦人/普通股的控制力**（投票權、董事席次條款），用後續輪次管理組成。
- [ ] 補進有「**規模化 / 上市 / 該產業**」實戰的獨立董事。

### 2. 會前
- [ ] **會前 3–5 天寄出 board deck**；會議不是用來簡報。
- [ ] deck 結構：**關鍵指標儀表板 → 上次行動項追蹤 → 本期重點 → 1–2 個要他們幫忙決策的策略議題**。
- [ ] 指標**每次用同一套**，讓董事看得出趨勢。

### 3. 會中
- [ ] 先**快速**過治理事項 + 指標總覽（少量時間）。
- [ ] **大部分時間留給 1–2 個策略議題**的真實討論/決策。
- [ ] 明確記錄**行動項 + 負責人 + 期限**。

### 4. 會外（其實更重要）
- [ ] **No surprises**：壞消息**會前個別先講**，別讓董事在會議上第一次聽到。
- [ ] 跟每位董事維持 **1:1**（尤其最難搞的、最有幫助的）。
- [ ] 把董事**當資源用**：找高管/客戶、募資背書、產業 intro、把關。

### 5. 難搞董事 / 隨規模演化
- [ ] 衝突的董事：先 1:1，再靠 lead investor / 其他董事調解。
- [ ] 有**明確機制調整董事會**（後續輪次重設、買回、加獨立席次）。
- [ ] 隨規模把董事會從「**營運參與**」轉成「**治理 + 策略**」。

### 心態
- 董事會是**工具不是老闆**；CEO 設議程、主導敘事。
- 對指標**誠實透明**——藏壞消息最傷信任。

## 🔭 類比：這套框架 vs「Elon 怎麼變成現在這樣」

> ⚠️ 誠實聲明：你給的影片（YouTube `62X1vm3QBuE`）被沙箱擋住、我**無法觀看**，所以以下是「**書的框架 ↔ Elon 的普遍軌跡**」的概念性對照，**不是針對該影片逐段對應**。把影片的章節/重點貼給我，我可以做精準 mapping。

**結論：框架層次上類比成立——但最有意思的是 Elon 同時「體現」也「違反」這本書。**

| 書的原則 | Elon 體現它的地方 | Elon 違反它的地方 |
|---|---|---|
| CEO 要**自我顛覆**、每階段升級 | Zip2/X.com → Tesla/SpaceX，從年輕創辦人變「工程+製造+資本」全能營運者 | — |
| **招能帶你下一階段的高管、其餘授權** | SpaceX 的 **Gwynne Shotwell（總裁/COO）**＝教科書級：Elon 顧願景/工程，她顧營運/商務/客戶 | 但在多數產品/工程上他**極度集中、少授權**，與書中「放手靠系統」相反 |
| **保留創辦人控制 + 慎管董事會** | 多家公司維持高控制力；收購 Twitter 後**直接重組董事會與組織**（書中「reorg 要快」的極端版） | Tesla 薪酬/董事獨立性爭議＝書中「董事會治理」的反面教材 |
| **願景重複講很多次** | 火星殖民、永續能源——把宏大敘事講到全世界都會背 | — |
| **後期募資 + secondary、估值紀律** | SpaceX 多輪私募 + 員工套現；**刻意不急 IPO（控制 > 流動性）** | — |
| **訂極高標準 + 動作要快** | 第一性原理、瘋狂節奏、把成本/規格重訂 | 節奏常以**人員耗損**為代價，書中強調「有尊嚴地管理人」 |
| 找到 PMF 後**重心轉通路/分發** | 特斯拉用品牌/直營/超充網路當分發護城河 | — |

**一句話**：這本書是「**通用、可複製的專業管理 playbook**」；Elon 是「**高變異、走極端的同一組動力學案例**」。他在「**招世界級營運者補位 + 死守控制權 + 重複願景**」上完全照書走；但在「**集中決策、壓榨節奏、董事會治理**」上故意打破書的規則。所以拿這本書當尺，剛好能量出 Elon 哪裡是「最佳實踐」、哪裡是「高風險的個人賭注」。
