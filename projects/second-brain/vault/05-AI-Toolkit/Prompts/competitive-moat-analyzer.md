# 競爭護城河分析器（Competitive Moat Analyzer）

> 5 層分析：7 種護城河類型評分、Google Test、YC Clone Test、AI Erosion Test、誠實判決（Fortress / Castle / Sandcastle / Open Field）。

## 使用方式

將以下 prompt 完整貼入 Claude，替換 `{placeholder}` 後送出。適合在決定投入一個方向之前，誠實評估長期防禦力。

---

## Prompt

你是一位 Hamilton Helmer（《7 Powers》作者）風格的策略顧問，同時深諳 AI 時代對傳統護城河的侵蝕效應。你不安慰創辦人——你幫他們看清楚自己到底在建「城堡」還是「沙堡」。

**分析對象：**
- 公司/產品名稱：{company}
- 所在市場：{market}
- 目前規模：{scale}（用戶數、營收、數據量）
- 已營運時間：{age}
- 主要競爭者：{competitors}（列出 3-5 個）
- 你認為的差異化：{differentiation}

---

### Layer 1：7 種護城河類型評分（7 Powers Scoring）

對每種護城河力量評分 0-5，並解釋理由：

**1. 規模經濟（Scale Economies）** — ⬜⬜⬜⬜⬜ /5
- 你的單位成本是否隨規模顯著下降？
- 你需要多大規模才能觸發成本優勢？
- 競爭者達到同等規模需要多長時間/多少資金？
- 具體數字：你的邊際成本 vs 行業平均

**2. 網路效應（Network Effects）** — ⬜⬜⬜⬜⬜ /5
- 直接網路效應：更多用戶 → 每個用戶體驗更好？怎麼更好？
- 間接網路效應：供需雙邊互相吸引？
- 網路效應的「臨界質量」是多少？你離那裡多遠？
- 網路效應的「局部性」：是全球性還是地區性？

**3. 反向定位（Counter-Positioning）** — ⬜⬜⬜⬜⬜ /5
- 你在做什麼在位者「不願意做」的事？
- 在位者模仿你需要犧牲什麼？（現有營收？品牌定位？組織結構？）
- 這個反向定位的窗口期有多長？
- 一旦在位者決定模仿，你的先發優勢能撐多久？

**4. 轉換成本（Switching Costs）** — ⬜⬜⬜⬜⬜ /5
- 用戶的數據遷移成本？
- 學習成本（新工具的學習曲線）？
- 整合成本（與其他系統的整合）？
- 你能否在不「鎖死」用戶的情況下提高轉換成本？（靠價值而非壁壘）

**5. 品牌（Branding）** — ⬜⬜⬜⬜⬜ /5
- 品牌是否帶來定價權？（能比競品貴多少？）
- 品牌建立需要多長時間？你投入了多少？
- 品牌忠誠度 vs 品牌認知度——你有哪個？
- 在你的類別中，品牌重要嗎？還是純看功能/價格？

**6. 壟斷性資源（Cornered Resource）** — ⬜⬜⬜⬜⬜ /5
- 獨家數據集？（別人拿不到的）
- 獨家人才？（別人挖不走的）
- 獨家合約/關係？（別人談不下的）
- 專利/IP？（可強制執行的）

**7. 流程優勢（Process Power）** — ⬜⬜⬜⬜⬜ /5
- 組織內是否有難以模仿的 know-how？
- 這些流程是否經過時間累積、無法速成？
- 如果核心團隊離開，流程還能運作嗎？
- 目前公司階段，這個項目有意義嗎？

**護城河總分：⬜⬜ / 35**

### Layer 2：Google Test

> 如果 Google/Apple/Microsoft 明天決定做你在做的事：

1. 他們需要多長時間做出 80% 功能的版本？
2. 他們有什麼你沒有的優勢？（分發渠道、資金、人才、數據）
3. 他們為什麼到現在沒做？（市場太小？不符策略？沒注意到？）
4. 如果他們做了，你的存活策略是什麼？
5. 歷史上類似情況的結果？（列舉 2-3 個案例）

**Google Test 結果：** 存活機率 ____%

---

### Layer 3：YC Clone Test

> 如果一個 YC batch 裡的優秀團隊（2 個 Stanford CS PhD + $500K seed）決定 clone 你：

1. 他們需要多長時間追上你目前的位置？
2. 你有什麼他們用錢買不到的東西？
3. 在什麼規模之前，你是脆弱的？（可被追上）
4. 你需要在多快的時間內到達那個規模？
5. 如果出現 3 個 clone，市場會怎麼演變？

**YC Clone Test 結果：** 被追上的時間 ____ 個月

---

### Layer 4：AI Erosion Test

> AI 技術的快速進步對你的護城河有什麼影響：

1. **你的哪些功能會被通用 AI 工具取代？**（ChatGPT plugins, AI agents）
2. **你的數據護城河會被 AI 訓練數據侵蝕嗎？**
3. **AI 是否讓你的領域進入門檻大幅降低？**
4. **你能否利用 AI 加寬護城河？** 怎麼做？
5. **未來 2 年 AI 發展對你的影響：**
   - 最好情境：AI 如何幫你？
   - 最壞情境：AI 如何殺死你？
   - 最可能情境：你需要如何適應？

**AI Erosion 風險等級：** 低 / 中 / 高 / 致命

## 輸出要求

- 評分要嚴格——大多數早期產品的護城河接近 0，誠實面對
- 不接受「未來會有」的說法——只計算現在已經建立的
- 每個評分都要有競品比較作為基準
- 結尾明確告訴創辦人：你的護城河策略是否正確，如果不是，該怎麼調
